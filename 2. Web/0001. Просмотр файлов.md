**Задание:**

Нужно прочитать содержимое файла, который находится в запрещенной папке.

Адрес: http://31.207.77.216:4013

---
**Решение:**

Имеем классическое задание с уязвимостью path traversal. Эта уязвимость позволяет читать произвольные файлы на сервере, на котором запущено приложение.

Способы обхода:

- Использование относительного пути.
- Использование точек для поднятия вверх по директориям.
- Использование кодирования или специальных символов.
- Обход через символические ссылки.
- Null byte injection и др.

Прочитаем флаг с использование символической ссылки на текущую рабочую директорию процесса:

```text
 /proc/self/cwd/forbidden/flag.txt
```

Судя по формату флага именно такое решение задумывал автор, однако в данной задаче вариантов для обхода очень много:

```text
forbidden/flag.txt
./forbidden/flag.txt
allowed/../forbidden/flag.txt
allowed/../../app/forbidden/flag.txt
allowed//..//..//app/forbidden/flag.txt
//app/forbidden/flag.txt
../app/forbidden/flag.txt
..//app/forbidden/flag.txt
app/../forbidden/flag.txt
app/..//forbidden/flag.txt
```

Помимо файла с флагом можно собрать и информацию о системе. С помощью данной уязвимости удалось получить следующие файлы:

- /proc/self/root/etc/passwd.
- /proc/self/root/etc/hostname.
- /proc/self/root/etc/hosts.
- /proc/self/root/etc/os-release.
- /proc/self/cwd/requirements.txt.
- /proc/self/root/etc/services.

В данной задачи они мне не понадобились, но в более сложных заданиях с помощью данной уязвимости можно собрать информацию для дальнейшей атаки.
